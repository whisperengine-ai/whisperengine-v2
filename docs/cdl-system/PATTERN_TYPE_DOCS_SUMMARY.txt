## üìö Key Concepts Explained

### What is pattern_type?
A **categorical field** (`VARCHAR(50)`) in the `character_communication_patterns` database table that classifies communication behaviors.

**Examples**: humor, explanation, emoji, communication_style, voice_tone, metaphor

### How are they processed?

1. **Load** from PostgreSQL database
2. **Group** by pattern_type
3. **Format** into prompt sections
4. **Send** to LLM
5. **LLM generates** response reflecting patterns

### Your ARIA Example

```yaml
pattern_type: communication_style          ‚Üê Classification
pattern_name: manifestation_emotion        ‚Üê Specific behavior
pattern_value: Holographic appearance...   ‚Üê Description
context: all_contexts                      ‚Üê Always relevant
frequency: constant                        ‚Üê Always apply
```

**Result**: ARIA describes her holographic form reflecting emotions in every response.

### Frequency Values
| Value | Meaning | Usage |
|-------|---------|-------|
| `constant` | Always apply | Core identity patterns |
| `often` | 70-80% of responses | Important patterns |
| `sometimes` | 30-50% of responses | Situational patterns |
| `rarely` | <20% of responses | Edge case patterns |

### Common Pattern Types
| Type | Purpose |
|------|---------|
| `humor` | How character uses jokes/wit |
| `explanation` | How character teaches |
| `emoji` | Emoji usage preferences |
| `communication_style` | Overall communication |
| `voice_tone` | Personality of voice |
| `metaphor` | Analogy preferences |
